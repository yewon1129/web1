CHP12. 부모자식과 목록

포함하는 태그는 부모태그, 포함되는 태그는 자식태그라고 한다. 목차를 만들 때 쓰는 태그 = li ~ /li (list의 앞자를 따온 것)
  • 1. Html
  • 2. Css
  • 3. Javascript

    • li의 부모태그가 ul ~ /ul tag이다
    • 1. Html
    • 2. Css
    • 3. Javascript
    • instagram
    • facebook
    • twitter
    선택한 후 tab키를 누르면 들여쓰기가 된다.

    만약에 정보의 수가 1억개라면? 수정하기가 무척이나 힘들 것이다. 넘버링하는 태그 : ol ~ /ol
    1. instagram
    2. facebook
    3. twitter
    ol : ordered list ul : unordered list

    CHP13. 문서의구조와 슈퍼스타들


    문서이름을 바꾸고 싶을 때 : title ~ /title
    문서를 읽는 방식을 바꾸고 싶을 때 : meta charset="방식"
    charset = character setting
    잘 사용하는 방식 = utf-8
    지금까지 배운 태그들은 운문이라면,
    title 과 meta charset은 운문을 설명하는 태그
    • 운문을 표현하는 태그는 body~/body라는 태그로 묶기로 약속했다.
    • 운문을 설명하는 태그는 head~/head라는 태그로 묶기로 약속했다.

    상당히 고위층 태그죠?
    이 중에서 가장 고위층 태그는 html~/html 이다.
    그리고 관용적으로 html 태그 위에는 이렇게 써준다.
    < !doctype html > = 이 문서는 html 문서야~라는 설명
